Help

Frequently asked questions about LatencyManager.

What does LatencyManager scan?

It scans the open Ableton Live set through AbletonOSC and reports plugin/device latency by track and plugin.

Why does the app say AbletonOSC is offline?

Ableton Live needs to be running, AbletonOSC must be installed and enabled as a MIDI Remote Script, and it must be reachable on 127.0.0.1:11000.

Why do I need to reload AbletonOSC?

Reloading makes Ableton Live pick up the latency export handler used by the dashboard. Run python3 app.py --reload-abletonosc once after installing or updating this tool.

Why are some devices shown as inactive or unknown?

AbletonOSC may not expose every state for every device type. The dashboard shows Unknown when the scan cannot confidently read the active state.

How should I use the latency numbers?

Start with the bottleneck track. Deactivating a device does not remove its PDC contribution; delete it or use Freeze and Flatten to remove that latency.

Where are scan results stored?

The latest scan is cached locally at ~/Library/Application Support/LatencyManager/abletonosc-latency-report.json.

Why does the current project name sometimes fail to appear?

Project detection is macOS-specific and depends on Automation permissions plus Live exposing the open set path. Grant Automation permission if prompted, or check System Settings → Privacy & Security → Automation.

Setup checklist

Complete these steps to start scanning latency.

Ableton Live

Running

Not detected. Open Ableton Live and load a project.

AbletonOSC

Reachable on port 11000

Not reachable. Install the AbletonOSC MIDI Remote Script into ~/Music/Ableton/User Library/Remote Scripts/, then enable it in Live → Settings → MIDI.

Latency export handler

Available

Not found. The /live/song/export/latency handler is missing. Make sure you are using an AbletonOSC build that includes the latency export patch, then run python3 app.py --reload-abletonosc.

macOS Automation permission

Granted

Not granted. When prompted, allow access so the app can detect the current Live project. You can also enable it in System Settings → Privacy & Security → Automation.
